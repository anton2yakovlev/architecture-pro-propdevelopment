# Таблица ролей и полномочий в Kubernetes

| Роль  | Права роли | Группы пользователей |
| --- | --- | --- |
| **cluster-admin** | Полный доступ ко всем ресурсам кластера, включая секреты, управление RBAC, создание и удаление namespace, управление nodes | Системные администраторы (admins), DevOps команда, SRE |
| **viewer-role** | Только чтение для основных ресурсов. Без доступа к секретам и управлению RBAC | Бизнес-аналитики, менеджеры проектов, QA-инженеры (viewers) |
| **developer-role** | Полный доступ к ресурсам приложений. Доступ на чтение к секретам в рамках своего namespace | Разработчики, DevOps-инженеры среднего уровня (developers) |
| **operator-role** | Управление кластерными ресурсами. Возможность управления namespace, но без доступа к RBAC | Операторы инфраструктуры, Platform Engineers (operators) |

## Описание групп пользователей

### 1. Группа "admins" (Системные администраторы)
- **Роль**: cluster-admin
- **Обязанности**: Полное управление кластером, настройка безопасности, управление доступами

### 2. Группа "viewers" (Наблюдатели)
- **Роль**: viewer-role
- **Обязанности**: Мониторинг состояния приложений, просмотр логов, анализ метрик

### 3. Группа "developers" (Разработчики)
- **Роль**: developer-role
- **Обязанности**: Разработка и деплой приложений, управление конфигурациями

### 4. Группа "operators" (Операторы)
- **Роль**: operator-role
- **Обязанности**: Управление инфраструктурой кластера, настройка хранилищ и сети
